# AWS Deployment Guide

This guide explains how to deploy your dating app to AWS using S3 and CloudFront.

## Architecture

- **Amazon S3**: Private bucket hosting the static web application files
- **Amazon CloudFront**: CDN delivering content globally with HTTPS
- **Origin Access Identity (OAI)**: Secure access from CloudFront to S3

## Prerequisites

1. **AWS CLI** installed:
   ```bash
   brew install awscli
   ```

2. **AWS Credentials** configured:
   ```bash
   aws configure
   ```
   You'll need:
   - AWS Access Key ID
   - AWS Secret Access Key
   - Default region (e.g., us-east-1)

3. **Build the application**:
   ```bash
   cd client
   npm install
   npm run build
   ```

## Deployment Steps

### 1. Initial Setup (One-time)

Run the setup script to create AWS infrastructure:

```bash
chmod +x deploy/*.sh
./deploy/aws-setup.sh
```

This script will:
- Create a private S3 bucket
- Block all public access to the bucket
- Create a CloudFront Origin Access Identity (OAI)
- Configure bucket policy to allow CloudFront access
- Create a CloudFront distribution
- Save configuration to `deploy/aws-config.env`

**Note**: CloudFront distribution deployment takes 15-20 minutes.

### 2. Deploy Application

After building your app, deploy it:

```bash
./deploy/deploy.sh
```

This script will:
- Upload all files from `client/dist/` to S3
- Set appropriate cache headers
- Create a CloudFront cache invalidation
- Display your CloudFront URL

### 3. Access Your Application

Your app will be available at the CloudFront domain:
```
https://d1234567890abc.cloudfront.net
```

The URL is displayed after setup and saved in `deploy/aws-config.env`.

## Updating Your Application

Whenever you make changes:

```bash
# 1. Build the app
cd client
npm run build
cd ..

# 2. Deploy
./deploy/deploy.sh
```

## Teardown (Remove All Resources)

To delete all AWS resources and stop incurring charges:

```bash
./deploy/teardown.sh
```

**Warning**: This will permanently delete:
- S3 bucket and all files
- CloudFront distribution
- Origin Access Identity

## Configuration Files

### aws-config.env

Generated by `aws-setup.sh`, contains:
- `BUCKET_NAME`: S3 bucket name
- `REGION`: AWS region
- `OAI_ID`: Origin Access Identity ID
- `CLOUDFRONT_ID`: CloudFront distribution ID
- `CLOUDFRONT_DOMAIN`: CloudFront domain name

**Important**: Do not commit this file to version control (already in .gitignore).

## Cost Estimation

### S3 Costs
- Storage: ~$0.023 per GB/month
- Requests: Minimal (CloudFront caches content)

### CloudFront Costs
- Data transfer: First 1 TB/month is $0.085/GB
- Requests: $0.0075 per 10,000 HTTPS requests
- Free tier: 1 TB data transfer out, 10M HTTPS requests/month for 12 months

**Estimated monthly cost**: $1-5 for low traffic, $10-50 for moderate traffic.

## Security Features

1. **Private S3 Bucket**: No public access allowed
2. **Origin Access Identity**: Only CloudFront can access S3
3. **HTTPS Only**: All traffic redirected to HTTPS
4. **Bucket Policy**: Restricts access to CloudFront OAI only

## Troubleshooting

### CloudFront shows 403 Forbidden
- Wait for distribution to fully deploy (15-20 minutes)
- Check bucket policy includes correct OAI
- Verify files exist in S3 bucket

### Changes not appearing
- Run `./deploy/deploy.sh` to invalidate cache
- Wait 2-5 minutes for invalidation to complete
- Hard refresh browser (Cmd+Shift+R)

### AWS CLI errors
- Verify credentials: `aws sts get-caller-identity`
- Check IAM permissions for S3 and CloudFront
- Ensure region is correct in aws-config.env

## Custom Domain (Optional)

To use your own domain:

1. **Request SSL certificate** in AWS Certificate Manager (us-east-1 region)
2. **Update CloudFront distribution** with custom domain and certificate
3. **Create DNS records** pointing to CloudFront domain

## CI/CD Integration

For automated deployments, use these scripts in your CI/CD pipeline:

```yaml
# Example GitHub Actions
- name: Deploy to AWS
  run: |
    ./deploy/deploy.sh
  env:
    AWS_ACCESS_KEY_ID: ${{ secrets.AWS_ACCESS_KEY_ID }}
    AWS_SECRET_ACCESS_KEY: ${{ secrets.AWS_SECRET_ACCESS_KEY }}
```

## Support

For issues:
1. Check CloudFront distribution status in AWS Console
2. Verify S3 bucket contents
3. Review CloudFront access logs (if enabled)
4. Check AWS CLI version: `aws --version`
